## Checklist de Lanzamiento

### Pre-lanzamiento (1 semana antes)
```
[ ] Instalador final subido a GitHub Releases / hosting
[ ] URL de descarga en [[08-web/08-06-pagina-descarga]] apunta al archivo real
[ ] Sistema de pago configurado y probado con compra real
[ ] Email de soporte activo
[ ] Páginas legales completas
[ ] Política de devolución publicada (30 días de garantía)
[ ] Flujo completo probado: comprar → clave → activar → usar
```

### Lanzamiento
```
[ ] Directorios de software: softonic.es, uptodown.com, malavida.com
[ ] Foros de hostelería y autónomos
[ ] Publicación en LinkedIn explicando el proyecto
[ ] Primer post en redes sociales
```

### Post-lanzamiento (primeras 4 semanas)
```
Semana 1-2: Monitorizar Search Console, Analytics, primeras ventas
Semana 3-4: Publicar segundo artículo de blog (según preguntas recibidas)

Métricas objetivo:
- Visitas a /descargar: 50+ en el primer mes
- Conversión demo → compra: 5-10%
- Posición Google "tpv verifactu": Top 50 en 30 días
```
